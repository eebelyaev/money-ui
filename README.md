# money-ui

Веб-интерфейс к сервису **calculations** (REST API). Репозиторий бэкенда обычно лежит рядом: `[../calculations](../calculations)`.

## Стек

- React 18, TypeScript, Vite, React Router
- Прокси в dev: запросы к `/api`, `/health`, `/docs`, `/openapi.yaml` уходят на бэкенд (цель по умолчанию `http://127.0.0.1:8080`)

## Вход и API

**Основной режим:** на `/login` вводятся **телефон или логин (username)** и **пароль** (телефон в БД в `person.login`, логин в `person.username`, хэш в `password_hash`). Запрос `POST /api/v1/calculations/auth/login` с полями `identifier` и `password` возвращает JWT с активной ролью из `user_role` (при нескольких ролях выбор делает сервер, детерминированно). Далее клиент шлёт заголовок `Authorization: Bearer <token>`.

**Режим разработки** (блок на странице входа): без пароля — в `sessionStorage` попадают только `X-Person-Id` и `X-Role`, как раньше.

Сессия хранится в `sessionStorage` до выхода или закрытия вкладки. На бэкенде задайте **`JWT_SECRET`** (см. README сервиса calculations).

На странице **Администрирование** для банкира дополнительно выставляется `X-Admin-Context: 1` (см. `docs/ADMIN_CALCULATIONS.md`), чтобы API позволял полный обзор договоров в режиме админки.

## Маршруты UI

| Путь | Описание |
|------|----------|
| `/login` | Вход: телефон или username и пароль (или dev: person id и роль) |
| `/` | Зависит от роли (см. ниже) |
| `/admin` | Администрирование: участники, договоры, платежи (роли **banker** или **admin**) |
| `/contracts/:id` | Карточка договора: остатки по снимкам, периоды истории; для банкира/админа — пересчёт, закрытие, новый платёж |
| `/banker`, `/client` | Редирект на `/` |

**Главная `/` без входа** — краткая справка и ссылка на вход.

**После входа:**

- **client** — список своих договоров (`GET /me/contracts`), открытие договора, история остатков в строках таблицы.
- **banker** — список клиентов портфеля, открытые договоры, история остатков, переход в карточку договора.
- **admin** — та же главная, что у банкира (клиенты и договоры); вкладка **Администрирование** в меню ведёт на `/admin`.

Остальные пути ведут на `/`.

## Запуск

1. Поднять **calculations** (Postgres, миграции, HTTP на нужном порту, по умолчанию ожидается `8080` под прокси Vite).
2. В каталоге `money-ui`:

```bash
npm install
npm run dev
```

Откройте в браузере URL из вывода Vite (обычно `http://localhost:5173`).

Если API на другом хосте/порту:

```bash
VITE_API_PROXY_TARGET=http://127.0.0.1:9090 npm run dev
```

## Сборка

```bash
npm run build
```

Статика в `dist/`. В продакшене раздавайте её как статические файлы и настройте reverse-proxy: те же префиксы `/api`, `/health`, `/docs` и т.д. — на сервис calculations (либо включите CORS на бэкенде).

Клиент ходит в API по префиксу **`/api/v1/calculations`** (см. `src/api/client.ts`).
